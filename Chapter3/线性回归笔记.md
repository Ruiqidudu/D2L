### 第1步：导入库

```python
import numpy as np
import torch
from torch.utils import data
from d2l import torch as d2l
from torch import nn
```

---

### 第2步：设置真实参数，生成合成数据集

```python
true_w = torch.tensor([2, -3.4])
true_b = 4.2
features, labels = d2l.synthetic_data(true_w, true_b, 1000)
```

#### synthetic_data 做了什么？
1. 随机生成特征 X（1000 个样本，每个 2 个特征）
2. 用真实公式 `y = Xw + b` 算出 y
3. 加一点随机噪声，模拟真实数据

**目的**：事先知道真正的 w=[2, -3.4], b=4.2，训练后看模型学到的参数是否接近，验证模型是否正确。

#### features 就是 x1, x2
```
features[0] = [x1, x2]     →  输入给模型
labels[0]   = [y]          →  真实值，用来和预测值比较
```

---

### 第3步：数据打包 — TensorDataset + DataLoader

```python
def load_array(data_arrays, batch_size, is_train=True):
    dataset = data.TensorDataset(*data_arrays)     # 把特征和标签打包成一对
    return data.DataLoader(dataset, batch_size, shuffle=is_train)  # 按批次取出

batch_size = 10
data_iter = load_array((features, labels), batch_size)
```

| 组件 | 作用 |
|------|------|
| `TensorDataset` | 把 features 和 labels 一一对应打包 |
| `DataLoader` | 按 batch_size 取出数据，支持打乱 |

对比手写版：

| 操作 | 手写版 (`08.py`) | 封装版 (`08_2.py`) |
|-----|-----------------|-------------------|
| 打包数据 | 手动用 indices 取 | TensorDataset |
| 分批 | 手动 yield | DataLoader |
| 打乱 | random.shuffle | shuffle=True |

---

### 第4步：定义模型 — nn.Sequential + nn.Linear

```python
net = nn.Sequential(nn.Linear(2, 1))
```

#### nn.Sequential
- 是一个**容器**，按顺序存放网络层
- 数据从第一层传入，依次经过每一层，最后输出
- 这里只有一层 `nn.Linear(2, 1)`，所以 Sequential 只包了这一层

#### nn.Linear(2, 1)

```python
nn.Linear(输入特征数, 输出特征数)
```

| 参数 | 值 | 含义 |
|------|-----|------|
| 输入特征数 | 2 | 每个样本有 2 个特征（x1, x2） |
| 输出特征数 | 1 | 输出 1 个预测值（y） |

数学公式：
```
y = x1·w1 + x2·w2 + b
```

`nn.Linear` 内部自动创建了两个参数：

| 参数 | 访问方式 | 形状 | 含义 |
|------|---------|------|------|
| weight | `net[0].weight` | (1, 2) | w1, w2 |
| bias | `net[0].bias` | (1,) | b |

**所以参数就是 w1, w2, b 这三个。**

---

### 第5步：参数初始化

```python
net[0].weight.data.normal_(0, 0.01)   # 用正态分布 N(0, 0.01) 随机初始化 w1, w2
net[0].bias.data.fill_(0)             # 偏置 b 初始化为 0
```

- `normal_(mean, std)`：用正态分布填充，每次运行值不同
- `fill_(val)`：全部填为固定值

---

### 第6步：定义损失函数

```python
loss = nn.MSELoss()
```

- **MSE** = Mean Squared Error（均方误差）
- 公式：`loss = (ŷ - y)² / 2`
- 接收两个参数：
  - 第一个：预测值 `net(X)` — 就是线性函数 ŷ = w1·x1 + w2·x2 + b
  - 第二个：真实值 `y`

---

### 第7步：定义优化器

```python
trainer = torch.optim.SGD(net.parameters(), lr=0.03)
```

- `net.parameters()` 返回网络中**所有需要训练的参数**
- 对于 `nn.Linear(2, 1)`，就是 weight（w1, w2）和 bias（b）
- 好处：网络层数多时，不需要手动列出每一层的参数

---

### 第8步：训练循环

```python
num_epochs = 3
for epoch in range(num_epochs):          # 外层：遍历整个数据集多次
    for X, y in data_iter:               # 内层：每次取一个 batch（10个样本）
        l = loss(net(X), y)              # 计算批次误差（10个样本的平均）
        trainer.zero_grad()              # 清空上一轮的梯度
        l.backward()                     # 反向传播，计算梯度
        trainer.step()                   # 更新参数 w1, w2, b
    l = loss(net(features), labels)      # 计算全体误差（1000个样本的平均）
    print(f'epoch {epoch + 1}, loss {l:f}')
```

#### 批次误差 vs 全体误差

| 位置 | 代码 | 数据量 | 用途 |
|-----|------|-------|------|
| 内层循环 | `loss(net(X), y)` | 10 个样本（batch） | 计算梯度，更新参数 |
| 外层循环 | `loss(net(features), labels)` | 1000 个样本（全体） | 打印查看训练效果 |

#### 为什么要用 batch 训练？

| 方式 | 每次用多少数据 | 特点 |
|------|--------------|------|
| 单个样本（SGD） | 1 个 | 更新频繁但不稳定，无法利用并行加速 |
| 小批量（Mini-batch） | 10~几百个 | ✅ **折中方案**，兼顾效率和稳定性 |
| 全部数据（Batch） | 全部 | 更新太慢，容易陷入局部最优 |

三个原因：
1. **内存限制**：数据太大时 GPU 放不下
2. **计算效率**：矩阵运算可以利用并行加速
3. **泛化能力**：随机性帮助跳出局部最优

## 八、关于反向传播的深入理解

### 1. 为什么不能直接代入新 x？

对于线性回归：
```
y = x1·w1 + x2·w2 + b
loss = (ŷ - y)² / 2
```

对 w1 求导的结果是：
```
∂loss/∂w1 = (ŷ - y) · x1
```

这个表达式里确实没有 y 以外的中间变量，看起来就是关于 x1, x2, w1, w2, b 的一个函数。

**但梯度公式里含有 w1, w2, b 这些参数，而参数是不断变化的！**

| 迭代次数 | w1 | w2 | b | 梯度值 |
|---------|-----|-----|-----|-------|
| 第1次 | 0.01 | 0.01 | 0 | 代入当前值计算 |
| 第2次 | 已更新 | 已更新 | 已更新 | 重新计算 |

梯度公式形式不变，但代入的参数值一直在变，所以每次迭代都需要重新计算梯度。

### 2. 多层网络中间变量消不掉

对于多层神经网络：
```
输入层 → 隐藏层1 → 隐藏层2 → 输出层 → loss
  x1       h1         h2         ŷ
  x2
```

loss 对 w1（第一层的权重）求导，链式法则展开为：
```
∂loss/∂w1 = ∂loss/∂ŷ · ∂ŷ/∂h2 · ∂h2/∂h1 · ∂h1/∂w1
```

**中间变量 h1, h2 不会消掉！** 每一层都有自己独立的参数和中间输出。

### 3. 反向传播的逐层计算过程

反向传播的精髓是**从后往前，逐层计算**：

```
第1步：∂loss/∂ŷ     ← 只需要 loss 和 ŷ
第2步：∂loss/∂h₂    ← 需要第1步的结果 + h₂ 的信息
第3步：∂loss/∂h₁    ← 需要第2步的结果 + h₁ 的信息
第4步：∂loss/∂w₁    ← 需要第3步的结果 + w₁ 的信息
```

每一层的梯度都依赖上一层的计算结果，这就是为什么叫"反向传播"——梯度像信号一样从输出端一层层往回传。

### 4. 链式求导公式是"固定模板"

链式求导的**公式结构**是固定的，不会变。每次迭代变化的只是**代入的具体数值**：

```
前向传播（计算数值）：
x → 算出 h₁ → 算出 h₂ → 算出 ŷ → 算出 loss
    ↑          ↑          ↑          ↑
  记录 w₁    记录 h₁     记录 h₂     记录 ŷ

反向传播（套用公式）：
loss ← 乘以 ∂loss/∂ŷ ← 乘以 ∂ŷ/∂h₂ ← 乘以 ∂h₂/∂h₁ ← 乘以 ∂h₁/∂w₁
    ↑          ↑          ↑          ↑          ↑
  得到梯度   得到梯度   得到梯度   得到梯度   得到 ∂loss/∂w₁
```

**前向传播算数值，反向传播套公式。** 每次迭代重复这个过程，参数就一步步更新到最优。

### 5. 总结

| | 线性回归 | 多层神经网络 |
|--|---------|-------------|
| 梯度公式 | 中间变量消掉了 | 中间变量**不会消掉** |
| 能否直接代入新 x | ✅ 可以手写公式 | ❌ 必须逐层反向传播 |
| 为什么还要用 backward() | 方便，自动计算 | 必须，手动算不了 |

反向传播不是"多此一举"，而是**解决多层网络梯度计算的唯一可行方法**。PyTorch 的 `backward()` 就是自动完成从后往前逐层传递的过程。

## 九、SGD 的学习率问题与 Adam 优化器

### 1. 问题：SGD 在最小值附近会震荡吗？

SGD 的更新公式：
```
w_new = w - lr · gradient
```

**步长 = 学习率 × 梯度大小**

| 位置 | 梯度大小 | 步长 | 效果 |
|------|---------|------|------|
| 远离最小值（陡坡） | 大 | 大 | 快速下降 ✅ |
| 鞍点/平坦区 | 接近 0 | 很小 | 靠随机性"晃"出去 ✅ |
| 最小值附近 | 小 | 小 | 慢慢逼近，不会大幅跨过 ✅ |

**越靠近最小值，梯度越小，步长也越小**，所以不会一步跨过最小值。

### 2. SGD 为什么能跨过鞍点？

鞍点的梯度也是 0，按理说也会停住。但 SGD 和梯度下降的关键区别在于：

**SGD 用的是小批量样本的梯度，不是全体样本的真实梯度。**

```
全体样本的真实梯度（在鞍点）= 0  →  停住 ❌
小批量样本的估计梯度（在鞍点）≈ 0  →  但带噪声，可能 ≠ 0 ✅
```

这个"噪声"就是 SGD 跨过鞍点的关键：
- 鞍点处：真实梯度 = 0，但小批量梯度 ≈ 0 + 噪声 → 可能把参数"推"出去
- 最小值处：真实梯度 = 0，小批量梯度 ≈ 0 + 小噪声 → 在最小值附近震荡，但不会跑远

### 3. 能不能检测到震荡就自动调小学习率？

**能！这就是 Adam 优化器的核心思想。**

#### Adam（Adaptive Moment Estimation）

Adam 维护了两个东西：

```
1. 梯度的一阶动量（梯度的指数移动平均）
   → 判断梯度方向是否一致

2. 梯度的二阶动量（梯度平方的指数移动平均）  
   → 判断梯度大小是否在震荡
```

**核心逻辑：**

| 情况 | 一阶动量 | 二阶动量 | Adam 的行为 |
|------|---------|---------|------------|
| 陡坡下降 | 方向一致，值大 | 值大 | 大步前进 ✅ |
| 最小值震荡 | 方向来回变，值≈0 | 值大（因为来回跳） | **自动缩小步长** ✅ |
| 鞍点平坦区 | 方向随机，值≈0 | 值小 | 保持步长，继续探索 ✅ |

#### 直观理解

想象你在下山：

| 优化器 | 行为 |
|--------|------|
| **SGD** | 固定步长，到谷底了还在原地踏步 |
| **SGD + 学习率衰减** | 每走一段路就把步长缩小一点（不管地形） |
| **Adam** | **智能步长**：发现你在原地踏步 → 自动缩小步长；发现你在爬坡 → 自动放大步长 |

#### 代码对比

```python
# SGD：固定学习率
trainer = torch.optim.SGD(net.parameters(), lr=0.03)

# Adam：自适应学习率
trainer = torch.optim.Adam(net.parameters(), lr=0.001)
```

**Adam 已经成为深度学习中最主流的优化器**，因为它：
1. ✅ 自动调节学习率，不用手动调
2. ✅ 对初始学习率不敏感（设 0.001 通常就够）
3. ✅ 收敛更快更稳

### 4. 各种优化器对比

| 优化器 | 学习率策略 | 特点 |
|--------|----------|------|
| **SGD** | 固定 | 简单，但需要手动调 lr |
| **SGD + Momentum** | 固定 + 惯性 | 加速收敛，减少震荡 |
| **SGD + 学习率衰减** | 按 epoch 缩小 | 死板，不管实际地形 |
| **Adagrad** | 自适应（累计梯度平方） | 学习率只会越来越小 |
| **RMSProp** | 自适应（指数移动平均） | 解决了 Adagrad 学习率归零的问题 |
| **Adam** | **自适应（动量 + RMSProp）** | **目前最常用，综合最好** |

### 5. 那为什么还要学 SGD？

既然 Adam 这么好，为什么教材还用 SGD？

1. **理解原理**：Adam 是 SGD 的改进，不懂 SGD 就不懂 Adam
2. **泛化能力**：某些任务（如 CV）SGD + 好的调参反而泛化更好
3. **可解释性**：SGD 的行为更简单，容易 debug

### 6. 总结

| 问题 | 答案 |
|------|------|
| 最小值处梯度大吗？ | 不大，梯度 ≈ 0 |
| 会一步跨过最小值吗？ | 不会，步长 ≈ 0 |
| SGD 怎么跨过鞍点？ | 靠小批量的随机噪声 |
| 能不能检测震荡自动调小学习率？ | ✅ 能，这就是 Adam |
| 为什么教材还用 SGD？ | 为了理解原理，且某些场景 SGD 更好 |
| 实际项目中用什么？ | **大部分用 Adam**，少数用 SGD + 精心调参 |
