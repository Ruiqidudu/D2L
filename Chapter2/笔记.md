## 一、pandas 基础操作

### 1. iloc — 按位置索引

`iloc` 全称 **integer location**，用数字下标定位数据。

```python
data.iloc[行, 列]
```

**示例：**
```python
data.iloc[:, 0:2]   # 所有行，第 0~1 列（前两列）
data.iloc[:, 2]     # 所有行，第 2 列
data.iloc[0, :]     # 第 0 行，所有列
data.iloc[1, 0]     # 第 1 行，第 0 列
```

**对比：**
| 方法 | 用啥索引 | 示例 |
|------|---------|------|
| `iloc` | 数字位置 | `data.iloc[:, 0]` → 第 0 列 |
| `loc` | 标签名 | `data.loc[:, 'NumRooms']` → 名为 'NumRooms' 的列 |

### 2. fillna — 填充缺失值

```python
# 用该列均值填充 NaN
inputs['NumRooms'] = inputs['NumRooms'].fillna(inputs['NumRooms'].mean())
```

**注意：** 不能对整个 DataFrame 计算均值，如果包含字符串列会报错。

### 3. get_dummies — 独热编码

将类别列转换为数值列（0/1）。

```python
# dummy_na=True 表示把 NaN 也当作一个类别处理
# dtype=int 让输出是 0/1 而不是 True/False
inputs = pd.get_dummies(inputs, dummy_na=True, dtype=int)
```

## 二、矩阵乘法的意义

### 核心公式

```
y = X @ w + b
```

- **X**：输入数据（多个样本）
- **w**：权重（模型要学习的参数）
- **b**：偏置
- **y**：预测结果

### 三个核心作用

| 作用 | 说明 |
|------|------|
| **线性变换** | 把原始特征组合成更高层的特征 |
| **批量计算** | 一次矩阵乘法 = 同时计算所有样本 |
| **层间传递** | 神经网络每层都是一次矩阵乘法 |

### 直观理解

```
输入 × 权重 = 输出
（原材料）（配方）（成品）
```

## 三、CPU vs GPU 并行计算

### 核心区别

| | CPU | GPU |
|--|-----|-----|
| 核心数 | 4~16 个 | 2000~10000+ 个 |
| 每个核心能力 | 很强（复杂逻辑） | 很弱（简单算术） |
| 适合任务 | 操作系统、浏览器 | 矩阵乘法、图像渲染 |

### 矩阵乘法的并行原理

```
C = A × B

C[0][0]  ← 核心0 负责
C[0][1]  ← 核心1 负责
C[0][2]  ← 核心2 负责
C[1][0]  ← 核心3 负责
...         ...
C[m][n]  ← 核心K 负责

所有核心同时开工，互不等待！
```

每个 `C[i][j]` 的计算相互独立，天然适合并行。

### 为什么 GPU 比 CPU 快几百倍？

- CPU = 10 个顶尖数学家，每人算 1000 道题
- GPU = 10000 个小学生，每人算 1 道题

同样算 10000 道简单算术，GPU 一瞬间完成。

### 深度学习离不开 GPU

> 用 GPU 训练 1 天的模型，用 CPU 可能要训练 **1 年**。

## 五、PyTorch 张量运算

### 1. 哈达马积（Hadamard Product）vs 矩阵乘法

| | 哈达马积 `*` | 矩阵乘法 `@` / `torch.mm` |
|--|-------------|-------------------|
| 运算方式 | 对应位置相乘 | 行 × 列 再求和 |
| 形状要求 | 形状相同（或可广播） | A的列数 = B的行数 |
| 例子 | (5,4) * (5,4) → (5,4) | (5,4) @ (4,3) → (5,3) |
| 用途 | 掩码、注意力权重 | 神经网络前向传播（核心） |

```python
A = [[1, 2],     B = [[5, 6],
     [3, 4]]          [7, 8]]

A * B（哈达马积）:          A @ B（矩阵乘法）:
[[1×5, 2×6],              [[1×5+2×7, 1×6+2×8],
 [3×7, 4×8]]               [3×5+4×7, 3×6+4×8]]
= [[5, 12],                = [[19, 22],
   [21, 32]]                  [43, 50]]
```

### 2. 矩阵向量积 `torch.mv` vs 逐元素乘 `*`

```python
A.shape      # (5, 4)
vec.shape    # (4,)
```

| 运算 | 结果形状 | 含义 |
|------|---------|------|
| `torch.mv(A, vec)` | (5,) | 真正的矩阵×向量（每行与vec做点积） |
| `A * vec` | (5, 4) | 广播后的逐元素乘（每列对应位置乘） |

**两者不能互相替代！**

### 3. axis 的含义（降维）

假设 A 是 5行×4列：

| 代码 | 结果形状 | 含义 |
|------|---------|------|
| `A.sum(axis=0)` | (4,) | 按列求和（每列一个数） |
| `A.sum(axis=1)` | (5,) | 按行求和（每行一个数） |
| `A.sum(axis=[0,1])` | 标量 | 所有元素求和 |

### 4. keepdims 的作用

**保持维度不丢失**，方便后续广播运算。

```python
A.sum(axis=1)                  # 形状 (5,)  — 变成一维向量
A.sum(axis=1, keepdims=True)   # 形状 (5, 1) — 还是二维

# keepdims 让广播能正常工作
A / A.sum(axis=1, keepdims=True)   # ✅ (5,4) / (5,1)
A / A.sum(axis=1)                   # ❌ (5,4) / (5,) 形状不匹配
```

> keepdims 是编程上的技巧，不是数学概念。数学上求和就是降维，但编程中保留维度壳子方便后续计算。

### 5. 范数

```python
u = torch.tensor([3.0, -4.0])

torch.norm(u)              # L2范数 = √(3²+4²) = 5.0
torch.abs(u).sum()         # L1范数 = |3|+|-4| = 7
torch.norm(torch.ones(4,9)) # Frobenius范数 = √(36×1) = 6
```

## 六、广播机制（Broadcasting）

### 1. 什么是广播

**自动把不同形状的张量对齐到相同形状，方便做逐元素运算。**

广播**没有真的复制数据**，只是在计算时"逻辑上"对齐形状。

### 2. 广播的规则

从最后一个维度往前对比，要么相等，要么一个是 1，要么缺失。

```
A.shape = (5, 4)
B.shape =    (4,)    → 对齐为 (1, 4) → 广播为 (5, 4)
✅ 语义：每个样本减去同一组均值

A.shape = (5, 4)
B.shape = (5, 1)    → 广播为 (5, 4)
✅ 语义：每行除以该行的和

A.shape = (5, 4)
B.shape = (3, 2)    → 不匹配
❌ 语义：两个无关矩阵相加没意义
```

### 3. 有意义的广播 vs 无意义的广播

| | 有意义的广播 | 无意义的广播 |
|--|-------------|-------------|
| 形状关系 | 一个维度是1，或形状部分匹配 | 完全不匹配 |
| 语义 | 一个数据是另一个的"统计量"或"标量" | 两个独立不相关的数据 |
| 例子 | `A - mean`、`A / sum`、`A + 1` | 价格矩阵 + 温度矩阵 |

### 4. 没有广播 vs 有广播

| 操作 | 没有广播（手动扩展） | 有广播（一行搞定） |
|------|-------------------|----------------|
| 所有元素 +1 | `A + torch.ones(5,4)` | `A + 1` |
| 每列减均值 | `A - mean.reshape(1,4).expand(5,4)` | `A - mean` |
| 每行除以和 | `A / sum_A.reshape(5,1).expand(5,4)` | `A / sum_A` |

> **广播是语法糖，让你只关心"数学上怎么算"，不用管"形状怎么对齐"。**

## 七、自动求导（Autograd）

### 1. grad_fn 是什么

当张量有 `requires_grad=True` 时，PyTorch 会记录所有运算，生成 `grad_fn`。

```python
x = torch.arange(4.0)
x.requires_grad_(True)

y = x * x       # y 有 grad_fn=<MulBackward0>
z = x + 1       # z 有 grad_fn=<AddBackward0>
```

`grad_fn` 表示"这个张量是怎么算出来的"，用于反向传播时求梯度。

### 2. torch.dot(x, x) vs x * x

```python
x = torch.tensor([0, 1, 2, 3], dtype=torch.float32)
```

| 运算 | 结果 | 含义 |
|------|------|------|
| `torch.dot(x, x)` | 标量 14 | 点积：对应位置相乘再求和 |
| `x * x` | 向量 [0,1,4,9] | 逐元素乘：对应位置相乘，不求和 |

**关系：** `torch.dot(x, x) = (x * x).sum()`

### 3. 为什么 backward() 要求标量

数学上梯度定义为**标量函数对向量求导**：

```
y = f(x)     y 是标量，x 是向量
梯度 = [dy/dx₁, dy/dx₂, ..., dy/dxₙ]
```

如果 y 是向量，`y 对 x 求导` 得到的是**雅可比矩阵（Jacobian）**：

```
y = [y₁, y₂, ..., yₘ]    雅可比矩阵 J：
x = [x₁, x₂, ..., xₙ]      J[i][j] = dyᵢ/dxⱼ
```

### 4. 向量 y 的 backward 技巧

```python
y = x * x              # 向量 [0, 1, 4, 9]
y.sum().backward()     # 先求和变成标量，再 backward
```

**原理：** `y.sum()` 的梯度 = y 每个分量对 x 的梯度之和

```python
y = x * x
y.sum() = x₀² + x₁² + x₂² + x₃²

dy.sum()/dx₀ = 2x₀    # 等价于分别求梯度再相加
dy.sum()/dx₁ = 2x₁
```

### 5. 梯度与初始值的关系

| | 说明 |
|--|------|
| 梯度公式 | 与初始值**无关**（如 y=x² 的梯度总是 2x） |
| 梯度数值 | 与初始值**有关**（代入不同 x 得到不同值） |
| 初始值影响 | 影响训练速度和最终结果 |

### 6. 总结

| 场景 | 方法 | 结果 |
|------|------|------|
| 标量 y → 向量 x | `y.backward()` | 梯度向量 |
| 向量 y → 向量 x | `y.sum().backward()` | 梯度之和（常用） |
| 向量 y → 向量 x | `jacobian(f, x)` | 雅可比矩阵（不常用） |

> 深度学习中 loss 本身就是标量，所以几乎只用 `y.backward()`。
