# 感知机与多层感知机（MLP）

## 一、从单层到多层

### 单层（Softmax 回归）
```
输入 → Linear(784, 10) → 输出
```

### 多层感知机（MLP）
```
输入 → Linear(784, 256) → 激活函数(ReLU) → Linear(256, 10) → 输出
                              ↑
                        关键在这里！
```

### 为什么需要激活函数？

如果没有激活函数，两层线性层叠在一起其实还是**一层**：

```
Linear(784, 256) + Linear(256, 10) = Linear(784, 10)
```

因为线性变换的复合还是线性变换，加再多层也没用。

激活函数（如 ReLU）的作用就是**引入非线性**：

```
ReLU(x) = max(0, x)
```

这样两层就不再等价于一层了，模型能学到更复杂的模式。

### 直观理解

```
单层：只能画直线分类（线性决策边界）
多层：可以画曲线分类（非线性决策边界）
```

- **单层** = 只能用直尺画线分割
- **多层** = 可以用曲线任意分割

## 二、为什么不在神经元内部用非线性函数？

比如把 `wx + b` 改成 `w₁x² + w₂x + b` 这样的多项式？

### 1. 参数爆炸

如果每个神经元都用 `w₁x² + w₂x + b`：

```
单层（784→10）：
原来：784×10 = 7840 个参数
改成 x²：每个连接要学 w₁ 和 w₂ 两个参数 → 参数翻倍
```

如果网络更深，参数会爆炸式增长。

### 2. 表达能力其实没变强

理论上，**多层线性 + 简单激活函数** 和 **单层多项式** 的表达能力是等价的。

一个两层网络（Linear + ReLU + Linear）已经可以逼近任意函数了（通用近似定理）。而用 x² 的神经元，本质上还是在做多项式拟合，并不比 ReLU 更强。

### 3. 优化更困难

```
f(x) = w₁x² + w₂x + b
∂f/∂w₁ = x²
∂f/∂w₂ = x
```

梯度里出现了 x²，如果 x 很大（比如像素值 255），梯度会非常大，训练不稳定。

而 ReLU 的梯度要么 0 要么 1，非常稳定。

### 总结

| 方案 | 问题 |
|------|------|
| 神经元用 x² | 参数多、梯度不稳定 |
| **线性 + ReLU + 多层** | **参数少、梯度稳、效果好 ✅** |

> 简单的东西堆叠起来，效果反而更好。这就是深度学习"深"的意义所在。

## 三、激活函数

### 常见的激活函数

| 激活函数 | 公式 | 特点 |
|---------|------|------|
| **ReLU** | max(0, x) | 最常用，简单高效 |
| Sigmoid | 1/(1+e⁻ˣ) | 用于二分类输出层 |
| Tanh | (eˣ-e⁻ˣ)/(eˣ+e⁻ˣ) | 输出范围 [-1, 1] |

### 为什么 ReLU 比 Sigmoid 好？

#### 1. Sigmoid 的问题：梯度消失

```
Sigmoid(x) = 1/(1+e⁻ˣ)
Sigmoid'(x) = Sigmoid(x) × (1 - Sigmoid(x))
```

当 x 很大或很小时：

```
x = 5:   Sigmoid = 0.993,  梯度 = 0.993 × 0.007 ≈ 0.007  ← 几乎为 0！
x = -5:  Sigmoid = 0.007,  梯度 = 0.007 × 0.993 ≈ 0.007  ← 几乎为 0！
```

**梯度接近 0 → 参数几乎不更新 → 网络学不动了**

这就是著名的**梯度消失**问题。深层网络用 Sigmoid，前面几层基本学不到东西。

#### 2. ReLU 的梯度

```
ReLU(x) = max(0, x)
ReLU'(x) = 0 (x<0) 或 1 (x>0)
```

梯度要么 0 要么 1，**不会变小**，深层网络也能正常训练。

#### 3. "丢信息"其实是好事

ReLU 把负数变 0 确实丢了信息，但这不是坏事：

```
Sigmoid:  -5 → 0.007,   -1 → 0.27,   0 → 0.5
ReLU:     -5 → 0,       -1 → 0,       0 → 0
```

Sigmoid 保留了"负多少"的信息（-5 和 -1 输出不同），但这些信息有用吗？

在神经网络中，负数通常表示"这个特征不相关"，ReLU 直接清零，相当于**强制让网络只关注正相关的特征**，反而让网络更专注。

#### 4. 稀疏性

ReLU 会让大约 50% 的神经元输出 0（稀疏激活），这其实是优点：
- 计算更快（0 不用算）
- 减少过拟合（更少特征在起作用）
- 更接近生物神经元（大脑也是稀疏激活的）

### 对比总结

| | Sigmoid | ReLU |
|--|---------|------|
| 梯度范围 | 0~0.25，容易消失 | 0 或 1，稳定 ✅ |
| 负数处理 | 保留信息（但没用） | 直接清零（更专注）✅ |
| 计算量 | 指数运算，慢 | 一次比较，快 ✅ |
| 深层网络 | ❌ 梯度消失，训练不动 | ✅ 可以堆很多层 |
| 输出范围 | (0, 1) 有界 | [0, ∞) 无界 |

### 一个直观例子

想象你在分辨"是不是衬衫"：

```
Sigmoid 神经元：
"有领子"特征 = 0.9  → 可能是衬衫
"没领子"特征 = 0.1  → 可能不是
"有袖子"特征 = 0.8  → 可能是衬衫
"没袖子"特征 = 0.2  → 可能不是
→ 所有特征都保留，但梯度小，学得慢

ReLU 神经元：
"有领子"特征 = 9    → 强信号 ✅
"没领子"特征 = 0    → 忽略
"有袖子"特征 = 8    → 强信号 ✅
"没袖子"特征 = 0    → 忽略
→ 只保留正信号，梯度大，学得快 ✅
```

> ReLU 不是"丢信息"，而是**过滤掉噪声，放大有效信号**。

## 四、与生物神经元的联系

```
生物神经元：
树突 → 接收信号 → 细胞体处理 → 超过阈值就通过轴突输出信号
                                            ↓
                                   输出是"有"或"无"（放电/不放电）
                                   但放电频率可以不同（强弱信号）
```

人工神经元（ReLU）：

```
输入 x → wx + b → ReLU(max(0, x)) → 输出
                                    ↓
                            x > 0 就输出（放电）
                            x ≤ 0 就不输出（不放电）
                            x 越大输出越强（频率越高）
```

ReLU 和生物神经元惊人的相似：
- **x ≤ 0** → 输出 0（不放电）
- **x > 0** → 输出 x（放电，越强信号越大）

Sigmoid 更像另一种生物神经元：
- **x 很小** → 输出接近 0（几乎不放电）
- **x 很大** → 输出接近 1（饱和放电）
- **中间** → 平滑过渡

```
生物神经元：    ----▁▂▃▄▅▆▇██▇▆▅▄▃▂▁----  放电率
Sigmoid：       ----▁▂▃▄▅▆▇████████----  输出 0→1
ReLU：          ▁▁▁▁▁▁▁▂▃▄▅▆▇████████   输出 0→∞
```

> 深度学习确实是从生物神经元获得灵感的——**每个神经元接收输入，加权求和，然后决定是否"激活"输出信号**。无数个这样的简单神经元堆叠起来，就能识别出复杂的模式。

## 五、实验对比：ReLU vs Sigmoid

### 代码文件

| 文件 | 激活函数 | 说明 |
|------|---------|------|
| `10_Relu.py` | ReLU | MLP 标准实现 |
| `10_sigmoid.py` | Sigmoid | 对比实验，仅替换激活函数 |

### 训练结果

**ReLU（`10_Relu.py`）：**
```
epoch 1, loss 1.0404, train acc 0.6523, test acc 0.7439
epoch 5, loss 0.4562, train acc 0.8519, test acc 0.8063
epoch 10, loss 0.3864, train acc 0.8761, test acc 0.8420
epoch 15, loss 0.3459, train acc 0.8889, test acc 0.8517
epoch 20, loss 0.3203, train acc 0.8987, test acc 0.8628
epoch 25, loss 0.3013, train acc 0.9076, test acc 0.8535
```

**Sigmoid（`10_sigmoid.py`）：**
```
epoch 1, loss 1.7569, train acc 0.4431, test acc 0.5647
epoch 5, loss 0.6211, train acc 0.7790, test acc 0.7753
epoch 10, loss 0.5027, train acc 0.8278, test acc 0.8125
epoch 15, loss 0.4569, train acc 0.8449, test acc 0.8236
epoch 20, loss 0.4320, train acc 0.8538, test acc 0.8338
epoch 25, loss 0.4142, train acc 0.8589, test acc 0.8383
```

### 对比分析

| 指标 | Sigmoid | ReLU | 差距 |
|------|---------|------|------|
| 最终 loss | 0.4142 | **0.3013** | ReLU 低 27% |
| 训练准确率 | 85.9% | **90.8%** | ReLU 高 5% |
| 测试准确率 | 83.8% | **86.5%** | ReLU 高 3% |
| 第 1 个 epoch loss | 1.76 | **1.04** | Sigmoid 起步更慢 |
| 第 1 个 epoch 准确率 | 44.3% | **65.2%** | Sigmoid 起步慢很多 |

### 结论

1. **Sigmoid 起步明显更慢**
   - epoch 1 loss 1.76 vs ReLU 的 1.04
   - epoch 1 准确率 44% vs ReLU 的 65%
   - 这就是梯度消失的影响——Sigmoid 一开始学得很吃力

2. **Sigmoid 最终效果接近单层 Softmax**
   - 测试准确率 83.8%，跟之前单层 Softmax 回归（83.8%）几乎一样
   - 说明加了 Sigmoid 的隐藏层**没起到什么作用**，梯度消失导致隐藏层没学到东西

3. **ReLU 比 Sigmoid 高了 5 个百分点**
   - 这就是非线性的威力——ReLU 让隐藏层真正生效了
   - 而 Sigmoid 因为梯度消失，隐藏层相当于摆设

```
Sigmoid 的 256 个隐藏神经元 → 实际只有前面几层在工作，后面梯度 ≈ 0
ReLU 的 256 个隐藏神经元 → 全部正常工作，真正学到了非线性模式
```

## 六、实验对比：两层 vs 三层

### 代码文件

| 文件 | 网络结构 | 说明 |
|------|---------|------|
| `10_Relu.py` | 784 → 256 → 10 | 两层（1 个隐藏层） |
| `10_deeper.py` | 784 → 256 → 128 → 10 | 三层（2 个隐藏层） |

### 训练结果

**两层（`10_Relu.py`）：**
```
epoch 1,  loss 1.0404, train acc 0.6523, test acc 0.7439
epoch 5,  loss 0.4562, train acc 0.8519, test acc 0.8063
epoch 10, loss 0.3864, train acc 0.8761, test acc 0.8420
epoch 15, loss 0.3459, train acc 0.8889, test acc 0.8517
epoch 20, loss 0.3203, train acc 0.8987, test acc 0.8628
epoch 25, loss 0.3013, train acc 0.9076, test acc 0.8535
```

**三层（`10_deeper.py`）：**
```
epoch 1,  loss 1.8117, train acc 0.3477, test acc 0.5577
epoch 5,  loss 0.5288, train acc 0.8231, test acc 0.8081
epoch 10, loss 0.3935, train acc 0.8719, test acc 0.8512
epoch 15, loss 0.3394, train acc 0.8941, test acc 0.8550
epoch 20, loss 0.3076, train acc 0.9058, test acc 0.8690
epoch 25, loss 0.2778, train acc 0.9163, test acc 0.8588
```

### 对比分析

| 指标 | 两层（784→256→10） | 三层（784→256→128→10） | 差距 |
|------|-------------------|----------------------|------|
| 最终 loss | 0.3013 | **0.2778** | 三层低 8% |
| 训练准确率 | 90.8% | **91.6%** | 三层高 0.8% |
| 测试准确率 | 86.5% | **87.3%** | 三层高 0.8% |
| 第 1 个 epoch loss | 1.04 | 1.81 | 三层起步更慢 |
| 第 1 个 epoch 准确率 | 65.2% | 34.8% | 三层起步慢很多 |

### 结论

1. **起步更慢了**
   - epoch 1 准确率只有 34.8%（两层是 65.2%）
   - 因为网络更深，一开始所有层都是随机初始化，需要更多时间"热身"

2. **最终效果有小幅提升**
   - 训练准确率 91.6% vs 90.8%（+0.8%）
   - 测试准确率最高 87.3%（epoch 23），但波动较大
   - 说明多加一层确实学到了更多，但提升有限

3. **测试准确率波动大**
   ```
   epoch 12: test acc 0.8092  ← 突然掉到 81%
   epoch 13: test acc 0.8590  ← 又回到 86%
   epoch 20: test acc 0.8690  ← 最高
   epoch 25: test acc 0.8588  ← 又掉了
   ```
   测试准确率在 81%~87% 之间来回跳，说明**开始过拟合了**——模型记住了训练集的一些噪声，测试集上不稳定。

### 总结

```
两层（784→256→10）：    测试 86.5%  ✅ 够用
三层（784→256→128→10）：测试 87.3%  ⚠️ 提升不大，还容易过拟合
```

对于 Fashion-MNIST 这种简单数据集，**两层已经够了**。多加一层：
- 参数更多（256×128 + 128×10 = 34,048 个新参数）
- 训练更慢
- 容易过拟合
- 提升只有不到 1%
