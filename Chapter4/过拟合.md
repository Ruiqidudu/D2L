# 过拟合与欠拟合

## 一、什么是过拟合和欠拟合？

### 欠拟合（Underfitting）

模型太简单，连训练数据都学不好。

```
真实规律：y = 5 + 1.2x - 3.4x² + 5.6x³（三次曲线）
你的模型：y = wx + b（直线）
→ 怎么都拟合不了，训练 loss 很大
```

### 过拟合（Overfitting）

模型太复杂，把训练数据中的噪声也记住了。

```
真实规律：y = 5 + 1.2x - 3.4x² + 5.6x³
你的模型：y = w₀ + w₁x + w₂x² + ... + w₁₉x¹⁹（20 次多项式）
→ 训练集拟合得很好，但换测试集就完蛋
```

### 刚好（Good Fit）

模型复杂度匹配数据复杂度。

```
你的模型：y = w₀ + w₁x + w₂x² + w₃x³（三次多项式）
→ 训练集和测试集表现都好
```

## 二、实验：多项式回归（11.py）

### 代码思路

1. **造一个真实函数**：y = 5 + 1.2x - 3.4x² + 5.6x³（只有前 4 项非零）
2. **生成数据**：200 个随机 x，计算对应的 y，加一点噪声
3. **特征工程**：把每个 x 变成 20 维特征向量 [x⁰, x¹, x², ..., x¹⁹]
4. **三种情况对比**：
   - 只用前 2 个特征（欠拟合）
   - 用前 4 个特征（刚好）
   - 用全部 20 个特征（过拟合）

### 实验结果

**1. 欠拟合（只用 2 个特征）：**
```
训练 loss: 4.96  ❌  测试 loss: 35.22  ❌
学到的权重: [3.60, 3.03]
真实权重:   [5, 1.2, -3.4, 5.6]
```
- loss 巨大，400 个 epoch 完全没降下来
- 学到的权重跟真实值差很远
- 原因：只用 x⁰ 和 x¹，根本拟合不了三次曲线

**2. 刚好（用 4 个特征）：** ✅
```
训练 loss: 0.0096  ✅  测试 loss: 0.0083  ✅
学到的权重: [4.99, 1.20, -3.40, 5.65]
真实权重:   [5, 1.2, -3.4, 5.6]
```
- 训练和测试 loss 都很低，且接近
- 学到的权重跟真实值几乎一样
- 原因：正好用了 4 个特征，匹配真实函数的阶数

**3. 过拟合（用全部 20 个特征）：**
```
训练 loss: 0.010  ✅  测试 loss: 0.026  ⚠️
学到的权重: [4.99, 1.25, -3.44, 5.35, 0.13, 0.97, -0.07, ...]
真实权重:   [5, 1.2, -3.4, 5.6, 0, 0, 0, ...]
```
- 训练 loss 很低，但测试 loss 比刚好情况高 3 倍
- 前 4 项接近真实值，但后面 16 项全是噪声（本应为 0）
- 原因：模型用 20 个特征去拟合 100 个样本，把噪声也学进去了

### 对比总结

| 情况 | 训练 loss | 测试 loss | 权重 | 结论 |
|------|----------|----------|------|------|
| 欠拟合（2 特征） | 4.96 ❌ | 35.22 ❌ | 完全不对 | 模型太简单 |
| **刚好（4 特征）** | **0.010 ✅** | **0.008 ✅** | **几乎完美** | **✅ 最佳** |
| 过拟合（20 特征） | 0.010 ✅ | 0.026 ⚠️ | 多了很多噪声 | 模型太复杂 |

## 三、L2 正则化（权重衰减）

### 为什么需要 L2 正则化？

过拟合时，模型的权重会变得很大：

```
过拟合的权重：w = [100, -80, 120, -90, ...]
→ 输入稍微变一点，输出就剧烈变化
→ 曲线弯弯曲曲，正好穿过每个训练点
→ 但换测试集就完蛋
```

### L2 的原理

给 loss 加一个惩罚项，让权重尽量小：

```
新 loss = 原 loss + λ × (w₁² + w₂² + w₃² + ...)
          ↑              ↑
       预测误差      所有权重的平方和
```

λ（lambda）控制惩罚力度：
- λ = 0：没有正则化，可能过拟合
- λ 很小：轻微惩罚
- λ 很大：权重被压得很小，可能欠拟合

### 为什么权重小能防止过拟合？

```
权重很大：输入变一点，输出剧烈变化 → 曲线扭曲 → 过拟合
权重很小：输入变一点，输出只变一点 → 曲线平滑 → 泛化好
```

### 为什么强制缩小权重不会错过最小值点？

**因为原来的最小值点本来就不是好点。**

```
没有 L2：
训练集上的最小值：w₁=100, w₂=-80
→ 完美拟合训练集，但测试集一塌糊涂（过拟合）

加了 L2 后：
新最小值：w₁=0.5, w₂=-0.3
→ 训练集上差一点点，但测试集表现好很多（泛化）
```

L2 相当于把最小值点往原点（0, 0）方向拉，找到的"新最小值"虽然在训练集上差一点，但在测试集上更好。

### 偏差-方差权衡

| | 不加 L2 | 加 L2 |
|--|---------|-------|
| 偏差（训练集误差） | 低 ✅ | 高 ⚠️ |
| 方差（测试集波动） | 高 ❌ | 低 ✅ |
| 整体效果 | 过拟合 | 泛化更好 |

我们宁愿要**稳定地做对 85%**，也不要**训练集 100% 但测试集 50%**。

### 一个类比

想象你在考试：

```
没有 L2：你死记硬背了所有练习题（训练集）
→ 练习题全对 ✅
→ 但考试题稍微变一下就不会了 ❌

有 L2：你理解了核心方法，没死记硬背
→ 练习题错了几道 ⚠️
→ 但考试题也能做对 ✅
```

L2 就是**不让你死记硬背**，强迫你学"真正的规律"。

### 为什么函数一定会与约束圆有交点？

L2 的优化目标等价于：

```
在 w₁² + w₂² + ... ≤ C 的条件下，最小化 loss
```

也就是说：**给你一个预算 C，权重平方和不能超过 C，在这个范围内找 loss 最小的点。**

```
没有 L2：你可以走到山谷最低点（但可能在很远的地方）
有 L2：  你只能在圆形区域内活动，找区域内最低点
```

这个"区域内最低点"一定在**圆的边界上**（除非原最低点本身就在圆内），所以一定是 loss 等高线和圆的**切点**。

因为如果等高线和圆相交（不是相切），那沿着圆边界走还能找到 loss 更低的点，直到走到相切的位置。

```
相交（不是最优）：         相切（最优）：
    ○──○                     ○
   /    \                    /\
  /  ×   \                  /  \
  \      /                  \  /
   \────/                    \/
```

## 六、代码细节

### 损失函数

12.py 用的是 **MSELoss（均方误差损失）**：

```python
loss = nn.MSELoss(reduction='none')
```

公式：

```
MSELoss = (预测值 - 真实值)²
```

因为 12.py 做的是**线性回归**（预测一个具体的数值），所以用 MSELoss。

跟之前分类任务的区别：

| 任务 | 损失函数 | 输出 |
|------|---------|------|
| 分类（Fashion-MNIST） | CrossEntropyLoss（交叉熵） | 10 个类别的概率 |
| 回归（12.py） | MSELoss（均方误差） | 1 个具体数值 |

`reduction='none'` 表示不自动求平均，返回每个样本的 loss，代码里手动用 `l.sum() / l.numel()` 计算平均 loss。

### 真实函数

```python
true_w = torch.ones((num_inputs, 1)) * 0.01   # 200 个权重，全是 0.01
true_b = 0.05                                   # 偏置 0.05
```

所以真实函数是：

```
y = 0.01x₁ + 0.01x₂ + 0.01x₃ + ... + 0.01x₂₀₀ + 0.05
```

就是一个**200 维的线性函数**，每个特征的权重都是 0.01，偏置 0.05。

跟 11.py 的区别：

| | 11.py（多项式回归） | 12.py（L2 正则化） |
|--|-------------------|-------------------|
| 真实函数 | y = 5 + 1.2x - 3.4x² + 5.6x³ | y = 0.01x₁ + 0.01x₂ + ... + 0.05 |
| 输入维度 | 1 维（x）→ 特征工程变 20 维 | 200 维 |
| 训练样本 | 100 个 | **20 个** |
| 过拟合原因 | 特征太多（20 vs 4） | **样本太少（20 vs 200 维）** |

12.py 故意让训练集只有 20 个样本，但输入是 200 维，**样本数 < 特征数**，模型必然过拟合——它可以记住这 20 个样本的每个细节，因为参数（200 个 w）比样本（20 个）多得多。

### L2 范数在哪里使用

L2 范数在 12.py 里出现在两个地方：

**1. 训练时：`weight_decay` 参数**

```python
trainer = torch.optim.SGD([
    {"params": net[0].weight, 'weight_decay': wd},  # 这里！
    {"params": net[0].bias}], lr=lr)
```

`weight_decay=wd` 就是 L2 正则化。PyTorch 的 SGD 优化器内部会自动在 loss 里加上 `wd × (w₁² + w₂² + ...)`。

- `wd=0`：不加 L2，权重随便长
- `wd=3`：加 L2，权重被压小

注意 bias（偏置）没有加 weight_decay，因为偏置只是一个偏移量，不影响过拟合。

**2. 评估时：打印 L2 范数**

```python
print(f'w的L2范数（wd={wd}）：{net[0].weight.norm().item():.4f}')
```

`net[0].weight.norm()` 计算的就是权重的 L2 范数：

```
L2 范数 = √(w₁² + w₂² + w₃² + ... + w₂₀₀²)
```

这只是用来**观察**权重被压得多小，不参与训练。

### wd（weight_decay）的含义

`wd` 的全称是 **weight_decay（权重衰减）**，它控制 L2 正则化的强度。

PyTorch 的 SGD 优化器在更新权重时，实际做的计算是：

```
新 loss = 原 loss + wd × (w₁² + w₂² + ...)
```

所以 wd 就是 L2 惩罚项前面的**系数 λ**。

**wd 越大，惩罚越狠：**

```
wd = 0：   不加惩罚，权重随便长 → 容易过拟合
wd = 0.01：轻微惩罚
wd = 1：   中等惩罚
wd = 3：   强惩罚（12.py 用的）
wd = 10：  非常强的惩罚 → 可能欠拟合
```

**12.py 的对比：**

| wd | 权重 L2 范数 | 测试 loss | 效果 |
|----|------------|----------|------|
| **0** | **14.13** | **219.10** | ❌ 过拟合，权重很大 |
| **3** | **0.39** | **0.51** | ✅ 权重被压小，泛化好 |

**如果 wd 太大：**
```
wd = 100：
权重被压到几乎为 0 → 模型啥也学不到 → 欠拟合
```

**总结：**
```
wd = 0：  不限制权重 → 过拟合
wd 适中：限制权重 → 泛化好 ✅
wd 太大：权重太小 → 欠拟合
```

wd 就像调音量一样，太小听不清（过拟合），太大震耳朵（欠拟合），要找到合适的中间值。

### 为什么线性模型也会过拟合？

12.py 的模型就是一个纯线性层，没有 ReLU 激活函数：

```python
net = nn.Sequential(nn.Linear(num_inputs, 1))  # 没有 ReLU！
```

整个模型就是 `y = w₁x₁ + w₂x₂ + ... + w₂₀₀x₂₀₀ + b`，纯线性。

**过拟合的本质不是"模型非线性"，而是参数太多，样本太少。**

```
12.py：200 个参数（w₁~w₂₀₀ + b），只有 20 个训练样本
→ 参数是样本的 10 倍
→ 模型可以记住每个样本的细节
→ 即使没有 ReLU，也能过拟合
```

**线性模型怎么"记住"样本？**

想象只有 2 个样本、1 个特征：

```
样本 1: x=1, y=3
样本 2: x=2, y=5
```

线性模型 `y = wx + b` 有 2 个参数（w 和 b），2 个样本，刚好能解出唯一解：

```
3 = w·1 + b  →  w=2, b=1
5 = w·2 + b
```

但如果样本有噪声：

```
样本 1: x=1, y=3.1  (真实值 3，噪声 0.1)
样本 2: x=2, y=4.9  (真实值 5，噪声 -0.1)
```

模型会去拟合这个噪声，w 和 b 就会偏离真实值。

**12.py 更极端：** 200 个参数 vs 20 个样本，模型有无数种方式让训练 loss 为 0——它可以给每个样本分配一个"专属"的权重组合，把噪声也完美拟合。

**总结：**

```
过拟合的条件 = 模型容量 > 数据量
              ↑              ↑
           参数数量       样本数量
```

跟有没有 ReLU 没关系。12.py 虽然只有线性层，但 200 个参数远大于 20 个样本，所以照样过拟合。L2 正则化就是在这种情况下把权重压小，防止模型过度依赖某些特征去"死记硬背"样本。
